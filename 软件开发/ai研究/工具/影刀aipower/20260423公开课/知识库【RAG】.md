## RAG 背景

当前主流大模型并非万能，有无法回避的短板。

|分析维度|核心问题|具体表现|
|---|---|---|
|知识与时效性|知识停留在预训练数据的截止日期，无法实时更新|仅掌握公开数据，无企业 or 个人私有信息|
|认知与推理|事实幻觉，生成流畅但虚假矛盾的内容，语言流畅优先真实性|医疗、法律、金融等高精场景易产生错误误导|
|安全与可控|黑盒不可解释，风险不可控，决策过程不透明|高可信场景难以通过合规、监管审核|
|学习与迭代|无自主反馈闭环，无法从错误中自主修正，重复犯错|持续学习、增量更新难度大，需重训成本高|

我们可以给大模型添加外接知识库，让AI在生成前先去查相关信息再回答，这就是RAG技术。

## 1️⃣ 什么是RAG？

RAG 的全称是「Retrieval-Augmented Generation」（检索增强生成），就是给AI添加外接知识库，让它生成前先查相关信息再回答，避免编造内容。

## 3️⃣ Azure OpenAI vs 阿里云百炼

| 对比维度          | 阿里云百炼                                       | Azure OpenAI                                          |
| ------------- | ------------------------------------------- | ----------------------------------------------------- |
| **模型生态与技术底座** | 以通义千问系列模型为核心，自研向量模型，持续迭代优化                  | 以 OpenAI 系列模型为核心，技术迭代快，生态成熟                           |
| **多语言能力**     | 主打中文，同时支持英文等多语言，但英文理解与生成能力弱于 OpenAI         | 以英文为核心，多语言能力如英文、西语等行业领先，跨语言检索效果好                      |
| **检索增强适配**    | 对中文语义、行业术语、长文本的召回精度高，适配国内企业知识库场景            | 对英文、专业术语的理解精度高，适配海外知识库，中文场景表现一般                       |
| **数据合规与隐私**   | 数据存储与计算全流程在国内，符合国内数据安全法规，企业私有数据无需出境         | 支持国内 Azure 区域部署，但核心数据仍需遵循微软/OpenAI的全球数据政策，跨境数据流动需额外评估 |
| **成本与计费**     | 整体成本更低，向量模型、大模型调用单价均低于同级别 OpenAI 服务，无跨境带宽成本 | 调用单价更高，叠加跨境访问成本，长期大规模使用的整体成本远高于阿里云百炼                  |
| **持续迭代与更新**   | 模型更新节奏贴合国内业务需求，支持国内政策、行业动态的快速适配             | 模型迭代速度快，优先跟进海外前沿技术，但国内场景适配更新较慢                        |
