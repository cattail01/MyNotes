## 引子

### ai的本质

* ai是根据概率生成下一个token
* 多个token组合生成一句话

### 提示词介绍

* 是一种控制模型行为的方式

## 高质量prompt的核心：丰富、具体、少歧义

> **写好提示词的第一原则**：把AI当人看，不要指望AI猜你的心思，把它当个特别聪明但较真的实习生，把话说到位。

### 5个核心要素

#### 1. 角色定义

* 介绍：
	告诉AI它是谁，给AI定义一个最匹配任务的角色
* 示例：
	你是一名专业的「提示词工程师」。擅长根据用户提供的应用场景和具体需求描述，生成高质量、结构清晰、可直接复制使用的专业提示词。
* 原理：
	角色决定了 "视角"。专业的「提示词工程师」写的提示词，和普通用户写的提示词，逻辑、细节、输出质量完全不一样。你给AI定义一个角色，它就会调取那个角色的专业表达方式、思考框架和行业标准。

#### 2. 任务描述

- 介绍：
	告诉AI要做什么，对任务进行描述，具体要做什么事，越具体越好。
- 示例：
	根据用户提供的产品信息和产品图片，请你：
	1. 对产品图片进行分析，提炼出产品的核心卖点、目标用户、高频使用场景、差异化优势；
	2. 基于提炼结果，设计一套完整、可直接落地、符合Amazon平台标准的A+详情图设计方案；
	3. 设计方案需包含：整套详情图的产品核心信息、Amazon A+套图清单、风格与字体方案、配色方案。所有文案必须严格适配图片语言的规范，且整套详情图的清单数量必须与用户指定的生成数量完全匹配，不得随意增减。
- 具体到：
	* 具体到你自己都觉得 "这是不是太啰嗦了" 的程度。

#### 3. 输入参数

- 介绍：
	告诉AI要知道什么，给出与任务相关的参数。
- 示例：
	请为我生成{{Product_Name}}的A+详情图提示词，参数如下：
	1. 产品名称：{{Product_Name}}「告诉AI为哪款产品生成A+详情图提示词。」
	2. 产品名称：{{Product_Name}}「告诉AI为哪款产品生成A+详情图提示词。」
	3. 生成数量：{{Generation_Quantity}}「明确要求AI生成多少张详情图，避免多生成或少生成。」
	4. 详情图设计方案：{{Detailed_Image_Design_Plan}}「根据详情图设计方案，告诉AI每张图要讲什么卖点、用什么场景、按什么逻辑讲等。」
- 越多越好？
	- 不是，和本节点无关的参数别加。无关的参数，反而会稀释提示词，让AI抓不住重点。

### 4. 输出格式

* 介绍
	告诉AI以什么格式输出「如：格式要求、字数要求等」
- 标准
	- 明确 JSON 输出边界
	- 锁死顺序和数量
	- 强制字段完整性
	- 约束冗余内容
	- 其他（去ai化）
- 示例
	请按照以下要求输出：
	1. 格式：严格以JSON格式输出，不得添加任何解释、多余文字、Markdown格式或说明性内容，只返回纯JSON对象。
	2. 结构：必须按「主图、卖点图、场景图、细节图、参数图、竞品对比图、品牌/质量背书、多角度补充图、包装/全家福」的顺序生成图片提示词。
